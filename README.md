# 회의 일정 조율 — 핵심 플로우 프로토타입

토스 2026 디자인 챌린지. 본질 문제 **'재계산 비용'** 을 푸는 핵심 플로우 데모.
"빈 시간이 안 겹치면 멈추는" 기존 도구와 달리, **충돌 지점에서 '무엇을 옮기면 전원 가능'한 조정안을 계산 → 정중히 요청**까지 잇는다.

## 실행 방법 (코드 처음이어도 OK)

터미널에서 이 폴더로 이동한 뒤, 아래 두 줄만:

```bash
npm install      # 처음 한 번만 (이미 했으면 생략)
npm run dev      # 개발 서버 실행
```

→ 터미널에 뜨는 주소(보통 http://localhost:5173/)를 브라우저에서 열기.
중단은 터미널에서 `Ctrl + C`.

## 골든 패스 (이 순서로 클릭하면 핵심이 다 보임)

1. **0 개인 조건** — 점심 직후 회피·외근 같은 선호가 미리 등록됨 → `시작`
2. **1 회의 만들기** — 기본값(분기 전략 회의) 그대로, 참석자 6명 필참/선택 → `후보 시간 보기`
3. **2 후보 시간** — **"전원 가능: 없음"** 확인 (여기서 기존 도구는 멈춤) → `조정안 보기`
4. **3 조정안(★히어로)** — `화 14:00 — 김선우 팀 리뷰를 화 15:00로 이동` (추천) → `이 방법으로 요청`
5. **4 양보 요청** — 명분 카드 확인 → `김선우 님에게 보내기`
6. **5 응답** — 김선우 입장으로 전환됨 → `수락`
7. **6 확정** — 회의 화 14:00 확정 + 팀 리뷰가 15:00로 이동한 캘린더(초록 점선)

→ **5에서 `거절`** 하면 다음 조정안(박민지 양보 / 한지우 제외)으로 돌아간다.

## 폴더 구조
- `src/data/mock.ts` — 6명 캘린더·선호 (전원 공통 슬롯이 없도록 설계됨)
- `src/logic/scheduling.ts` — 후보 랭킹 + 조정안 생성 + 중요도 추정
- `src/screens/` — 6개 화면
- `npm run check` — 데이터가 의도대로(전원 슬롯 0, 조정안 3종)인지 콘솔 검증

## 아직 안 한 것 (의도적)
디자인 스타일(토스풍), 회의실 예약, 실제 캘린더 연동, 로그인, 모바일 — 다음 단계.
